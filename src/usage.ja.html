Usage

モニタリング

  • モニタリング画面を開くとモニタリングが始まります。全てのタブを対象にモニタリングを行ないます。
  • モニタリング中は、ブラウザの処理が重くなります。モニタリングが不要な場合は、モニタリングのタブを閉じるか、caputure チェックボックスのチェックを外してください。

アクセスログ

  • モニタリング画面に、取得したアクセスログをリスト表示します。
  • リストの行をクリックすると、 HTTP Header の詳細を表示します。
  • リストの行を右クリック(あるいはロングタップ)すると、ポップアップメニューが開きます

ポップアップメニュー

  • ポップアップメニューでは、選択しているログの URL リソースのダウンロード、表示などを行ないます
  • URL リソースのサイズが所定のサイズを越えている場合、URL リソース表示ができません。 URL リソースのサイズ制限は limit-size のスライドバーで変更してください。スライドバーの値は、新しく取得したアクセスログから有効になります。 アクセスログごとに、取得したリソースを保持するためその分メモリを消費します。
